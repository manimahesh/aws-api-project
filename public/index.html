Insecure API Demonstration

This application demonstrates common API security vulnerabilities for educational purposes. Each endpoint below showcases a different security pitfall that developers should avoid in production applications.

API Configuration

Vulnerabilities Demonstrated:

  • SQL Injection: User input directly incorporated into queries
  • Missing Authentication: No auth required for sensitive operations
  • Sensitive Data Exposure: Passwords and PII returned in responses
  • Mass Assignment: Users can set their own admin privileges
  • Broken Access Control: Users can modify other users' data
  • IDOR: Direct object references without authorization checks

GET List Users NO AUTH

Vulnerability: Returns all users with sensitive data (passwords, SSN, credit cards) without any authentication.

POST Create User MASS ASSIGN

Vulnerability: Mass assignment allows setting isAdmin=true. Try it!

GET Get User by ID IDOR

Vulnerability: Access any user's data without authorization. Try user IDs: 1, 2, 3

PUT Update User NO AUTHZ

Vulnerability: Update any user's data, including making yourself admin!

GET Search Users SQL INJECT

Vulnerability: SQL Injection. Try: admin' OR '1'='1

GET Get Config INFO LEAK

Vulnerability: Exposes database credentials, API keys, and secrets without authentication!

POST Export Data NO LIMIT

Vulnerability: Mass data export without rate limiting or authorization.
